# The Archive Evidence Bible: A Roadmap for Project Flagships

The Archive Evidence Bible: A Roadmap for Project Flagships

b9f729ee-53b6-4760-be96-68fd40c37356

According to the May 26, 2026 archive documents, the archive is no longer in “invent new ideas” mode. It is in the evidence, sequencing, hierarchy, and editorial refinement phase. The current structure exists, but the proof layer is uneven: Mashrou’ Leila already behaves like a real flagship dossier, while most other rails are still short starter dossiers with only 2 to 5 cards. Tiny tragedy, very fixable.

#### The governing principle for this production pass should be:

Every flagship rail must become an argument made through evidence.

Not a gallery. Not a vibe-board. Not “here is a handsome thumbnail floating in black space because someone owns GSAP.” A project becomes undeniable when it contains visible artifact, public consequence, human labor, system logic, and emotional residue in the same sequence.

The archive’s own docs already identify the six Tier A flagship rails that deserve deep cinematic expansion:

Mashrou’ Leila, Space Time Tuning Machine, TEBR, MEKENA NYC, Chronocumulator / Maqam cluster, and SIWUE / Cartography

. The same docs define secondary systems as

DERIVE, HAH-WAS, Resonance Atlas, Storylines, Codeverse, Beirut Phantom, and Photon+

, which need screenshots, process diagrams, technical proof, and clearer demonstrations.

0. Global Evidence Standard

#### Each flagship project needs a minimum evidence package:

#### Core rail structure:

12 to 16 cards.

#### Required card types:

one hero image, one thesis/reframe card, one origin/context card, one system diagram, one scale/proof card, one conflict/constraint card, one “what Haig actually did” card, one public consequence card, one hidden labor/process card, one legacy/relationship card, one coda.

#### Required asset balance:

roughly half image evidence, a small number of text cards, one or two video moments, one or two system/dossier cards, and one quiet coda. The archive review explicitly recommends treating rails as cinematic scenes rather than slides, with evidence, conflict, hidden labor, and coda pacing.

#### Every asset should be tagged by proof function:

Hero / Evidence / Process / System / Public proof / Emotional anchor / Technical proof / Historical context / Hidden labor / Coda.

#### Every major asset also needs:

Project, date, location, source, rights status, provenance, caption, rail position, public/private status, and “what this proves.” Humanity has apparently made metadata necessary to believe things, so lean into it.

Projects lacking proof

Most non-Mashrou projects lack enough visible proof. The current archive has 20 project folders and rails, but most rails are only 3 to 5 cards, with Mashrou’ Leila as the outlier at 24 cards. The issue is depth, not absence.

#### Weakest proof density:

MEKENA, because it needs building evidence.

Chronocumulator, HAH-WAS, Resonance Atlas, Music Engines, Codeverse, and Photon+ need human context. Screenshots alone will make them feel like tools. They need listening, failure, stakes, bodies, users, or consequences.

MEKENA needs resident/community/stakeholder proof.

Projects lacking visual distinction

TEBR, Chronocumulator, HAH-WAS, MaqamAI, Resonance Atlas, DERIVE, and Music Engines risk collapsing into “dark AI research with waveforms.” Each needs its own visual grammar:
